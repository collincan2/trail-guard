## Report Schema


This schema defines the structure and validation rules for hazard reports generated by the system.  
It ensures that all reports are complete, consistent, and usable.


## Allowed Hazard Categories

- Park Infrastructure
- Debris
- Animal
- Fire Hazard


## Required Fields

Every report must include:

- hazard_type: one of four, Debris, Animal, Park Infrastructure, or Fire Hazard
- severity_rating: a value of 1-5 depending on AI judgement (level of disruption)
- trail_segment: numerical value for pre-defined sectors of any particular park
- reported_timestamp: a HH:MM formatted time of when the image was uplaoded
- ai_description: string description generated by AI to describe issue/image
- recommended_action: string that defines what the AI gives as a soft recommendation

## JSON Schema

Can be found in validateschema.json. An example of a working/non-working code segment is shown below.  

## Working:

{  

  "hazard_type": "Debris",  
  
  "severity_rating": "4",  
  
  "trail_segment": "5",  
  
  "reported_timestamp": "10:05 AM",  
  
  "ai_description": "A fallen branch has blocked the path.",  
  
  "recommended_action": "Send crew to clear debris."  
  
}

## Non-Working:

{  

  "hazard_type": "Fallen Tree",  
  
  "severity_rating": "6",  
  
  "trail_segment": "North Park Loop",  
  
  "reported_timestamp": "yesterday",  
  
  "ai_description": null,  
  
  "recommended_action": null  
  
}  

Issues:
- Hazard type is not under the defined list
- Severity rating is a value 1 to 5
- Segment is a numerical ID, not a string literal
- Timestamp must be in HH:MM format
- AI Description and Recommended Action cannot be null.
